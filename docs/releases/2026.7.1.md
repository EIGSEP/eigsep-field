# 2026.7.1

_Release date: 2026-07-08_

Patch cut over `2026.7.0` for the July 2026 deployment: CMT VNA S11
data-transfer fixes plus a small observing-side VNA debugging feature.
No firmware, key-registry, sensor-schema, or systemd unit changes —
interface docs are identical to `2026.7.0`.

## What's new

- **eigsep-vna** (v1.4.1, bug fixes): the CMT R60 silently ignores
  Keysight-style `FORM:DATA REAL,64`, so binary transfers were never
  actually in effect — data transfer reverts to ASCII with S11 read via
  `CALC:DATA:SDAT?` (EIGSEP/CMT-VNA@c511cc7, @e4cd0a4); the driver now
  verifies instrument configuration on connect and re-pushes it on a
  cold start (EIGSEP/CMT-VNA@df1d506).
- **eigsep_observing** (v2.12.0): `vna_manual` gains one-off
  switch-path probes and a `--once` mode
  (EIGSEP/eigsep_observing@7ba15a5).

## Upgrade notes

- No re-image required: `eigsep-field sync-image` converges a flashed
  Pi to this release in place. Only the two Python packages change —
  no staged-file, unit, or firmware deltas.
- After the venv swap, restart services or reboot before deployment
  (`eigsep-field services list`).
