# 2026.7.0

_Release date: 2026-07-08_

First stable cut for the July 2026 deployment. Baseline for the
version-bump table is `2026.7.0b1`, the campaign's last pre-release
(`2026.4.0` was never published as a tag).

## What's new

- **eigsep_observing**: on-demand VNA sessions — `cmtvna.service` stays
  stopped except around an S11 measurement window
  (EIGSEP/eigsep_observing#188); tempctrl channels descope via a
  per-channel `installed` flag (#192); per-channel corr linear-range
  product in file headers and live status, with the deployed 2026-07-08
  product (#190, #196); az-only / el-only motor scans (#197).
- **picohost / Pico firmware**: tempctrl per-channel `installed` flag —
  firmware never samples a descoped channel, host exposes
  `set_installed` (pico-firmware v4.2.0).
- **eigsep-vna**: generic `measure_dut` for arbitrary switch paths
  (v1.4.0).
- **image**: `cmtvna.service` flipped to `on-demand` activation —
  dropped from `eigsep-panda.target`, sudoers grants `eigsep`
  passwordless start/stop, observe-side code owns the lifecycle (#53);
  `eigsep-field sync-image` in-place pre-deployment updater (#54).

## Upgrade notes

- No re-image required: `eigsep-field sync-image` converges a flashed
  Pi to this release in place. This is the first release with a
  published wheelhouse artifact, so the wheelhouse step performs a real
  venv swap instead of the mid-cycle skip.
- The cmtvna on-demand flip and eigsep_observing 2.11.0 must land
  together (an on-demand image with an older observe build never starts
  the service). This blessed tuple satisfies that by construction; do
  not cherry-pick one without the other on an already-flashed Pi.
- After the venv swap, restart services or reboot before deployment
  (`eigsep-field services list`).
