# 2026.7.0b1

_Release date: 2026-07-07_

**Beta.** Cut to exercise `eigsep-field sync-image` end-to-end on
deployed Pis (wheelhouse swap + venv reinstall from a Release asset).
Not a campaign release; the public website tag pin is deliberately
not bumped.

## What's new

- First release whose tag matches `v{manifest.release}` since the
  package pins moved, so `sync-image` can pull the wheelhouse from
  the Release asset instead of reporting "already at 2026.4.0".
- Ships the `eigsep-field sync-image` in-place updater (feat/sync-image,
  PR #54) on the image.

## Version bumps

Relative to the May 2026 rc images (`v2026.5.0-rc*`) deployed in the

new pins in its wheelhouse.

## Firmware

- Pico: `v4.1.0`

## Upgrade notes

On an already-flashed Pi (any May rc or the July beta):

```sh
cd ~/src/eigsep-field
git fetch --tags && git checkout v2026.7.0b1
sudo /opt/eigsep/venv/bin/pip install .
sudo eigsep-field sync-image
```
