# 2026.7.2

_Release date: 2026-07-09_

Final cut before the July 2026 field deployment. Headline feature is
the SP1 failsafe termination, which spans the pico firmware (GPIO 27
`sp1_term` command), picohost (`set_sp1_termination`), and
eigsep_observing (observing modes, VNA short/open sweeps, live-status)
— the pico UF2 bump and the observing bump are inseparable
(eigsep_observing pins `picohost>=4.5`). Sensor schemas changed
(`imu_az` descope, potmon `sp1_term`/`sp1_term_name`, peltier
`sensor_rejects`); interface docs are regenerated in this release. No
systemd unit changes across the whole span — staged units are
byte-identical to 2026.7.1's.

## What's new

- **eigsep_observing** (v2.14.0):
  - SP1 failsafe termination: observing modes, VNA short/open sweeps,
    live-status plumbing (EIGSEP/eigsep_observing#219).
  - Shipped SNAP config moves to `corr_acc_len = 2**26` (0.268 s tick)
    with `tapcp_timeout_s = 0.1`; TAPCP slow-transaction logging
    (EIGSEP/eigsep_observing#204).
  - `adc_stats` derived from ADC snapshots; dead register paths
    dropped. Live-status gains SNAP connector labels, physical wiring
    for the ADC pane, and a config panel with gating provenance.
  - Wiring manifest for this deployment (C000122, Rx modules) with
    authoritative module ids for the 4-chain roster.
  - `imu_az` azimuth descope — schema + live-status
    (EIGSEP/eigsep_observing#213); motor per-axis home
    (EIGSEP/eigsep_observing#205) and single-axis `motor_scan` sweeps.
  - Observer panda re-probing gated to a cooldown during outages,
    anchored at probe completion.
- **picohost / pico firmware** (v4.5.0):
  - SP1 failsafe termination on GPIO 27: `sp1_term` command in the
    potmon firmware, emulator state, and
    `PicoPotentiometer.set_sp1_termination` /  `sp1_term_name`
    (EIGSEP/pico-firmware#155).
  - tempctrl reports every plausible sample (rate guard is
    control-only) and recovers reporting after a sensor-trip latch;
    peltier wiring polarity fix.
  - el-only `calibrate-imu` with derived most-down home; az blend
    retired (EIGSEP/pico-firmware#152).
  - Whole-valued firmware floats published as floats
    (EIGSEP/pico-firmware#148).
- **eigsep_redis** (v2.3.2, bug fixes): TCP keepalive bounds
  half-open-peer hangs (EIGSEP/eigsep_redis@5ad352d); the first entry
  published after an absent-stream poll is now delivered
  (EIGSEP/eigsep_redis@526da80).

## Firmware

- Pico: `v4.2.0` → `v4.5.0` — **must be flashed in the field**
  (`flash-picos` / `eigsep-field revert pico-firmware`); sync-image
  only stages the blessed UF2.
- RFSoC bitstream: `v0.3.0` (unchanged, paired with eigsep_dac v0.3.0)

## Upgrade notes

- No re-image required: `eigsep-field sync-image` converges a flashed
  Pi in place (or stage the wheelhouse manually for the offline flow).
  No staged-file or unit deltas — this is a packages + pico-UF2
  release.
- The pico UF2 and the Python stack must move together: observing
  2.14.0's potmon schema requires firmware ≥4.5 stream fields
  (`sp1_term`, `sensor_rejects`). Flash the Picos as part of the same
  visit that swaps the venv.
- eigsep_observing v2.13.0 was tagged upstream but never published to
  PyPI (superseded by v2.14.0 within the hour); the manifest skips it.
- After the venv swap, restart services or reboot before deployment
  (`eigsep-field services list`).
